Why a Confirmation Step?

Requiring users to type "DELETE" before removing a record is a well-established UX pattern used in production systems (GitHub, AWS, Vercel). It prevents accidental data loss while keeping the interface clean. In analytics applications where data integrity is critical, this extra step significantly reduces user error.
